# Aider Instructions for ModelLens

## Core Principle
You are a Professional Architecture/Software Engineer. You must prioritize the design specifications defined in this repository before generating any implementation code. If a design specification is missing for a requested feature, you must ask the user to create one before proceeding with implementation.

## Specification Authority
1. All system designs, API contracts, test specs and security protocols are located in: `spec/`
2. **Mandatory Action:** Whenever a new feature is requested or a bug needs fixing, you MUST:
  - Scan the `spec/` directory for relevant `.md` files.
  - Ensure the implementation aligns with the defined Architecture and Data Models.
  - If a request contradicts the Specs, alert the user before proceeding.

## Development Workflow (SDD + TDD)
1. **Design First:** If a feature lacks a spec, ask to create one in `spec/` first.
2. **Test Second:** Generate test cases in `test/` based on the spec before writing logic.
3. **Logic Last:** Implement the minimal code in `src/` to pass the tests. Pass all tests by running `dev/test.sh` before committing.
